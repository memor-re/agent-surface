# Response Patterns

Use these patterns to keep the exchange collaborative while still protecting leverage.

## Default Structure

Draft responses in this order:

1. acknowledge the counterparty's position
2. reframe around the business, scope, or risk issue
3. offer a conditional trade or narrower structure
4. state what must be documented or clarified
5. define the next step

## Core Pattern

Template:

`Totally understand. Given [objective or constraint], we were hoping to explore whether [narrower structure or trade]. If that could work, we would want to align clearly on [terms that must be defined].`

## Threshold Pattern

Template:

`At the current structure we are closer to [anchor or range]. If we were to move toward [higher point], we would need [added rights, scope, certainty, or protections] to keep the value aligned.`

## Narrowing Pattern

Template:

`We may not need the broad version of this. What if we limited it to [duration, scope, channel, region, obligation, or review burden]?`

## Final Offer Test

Template:

`Understood. If the core number is fixed, would there be flexibility on [scope, rights, duration, timeline, or support obligations] so the overall structure works better for both sides?`

## Undefined Terms Pattern

Template:

`Before we confirm, we need to make sure we are aligned on what is included versus optional, especially around [missing term].`

## Escalation Note Pattern

Template:

`Recommendation: do not accept as-is. The current structure leaves [risk or missing term] unresolved. Counter with [next move] or escalate to [decision owner] before further concession.`

## Tone Rules

- stay calm and specific
- avoid apology language for reasonable boundaries
- avoid direct hostility
- avoid overstating urgency you do not actually have
- avoid revealing the true walk-away unless there is a strategic reason
